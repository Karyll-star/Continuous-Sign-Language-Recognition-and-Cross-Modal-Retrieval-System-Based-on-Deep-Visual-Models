## 概述

本项目提供两种"句子翻译"能力：

| 能力 | 方向 | 核心组件 | 增量训练方式 |
|------|------|----------|------------|
| 手语识别 | 图像帧 → 句子文本 | ResNet + 分类头 | `scripts/incremental_train.py` |
| 视频检索 | 文本句子 → 手语视频 | 句向量索引 + 余弦相似度 | `scripts/build_cecsl_index.py` |

两种方式适用场景不同：如果你想让模型**识别新的手语句子**（输入手语图片/视频，输出句子），使用方式一；如果你想让系统**根据中文句子推荐对应的手语视频**，使用方式二。

## 方式一：ResNet 手语识别模型增量训练

### 原理

已有模型是一个分类器：ResNet 骨干提取图像特征，分类头将特征映射到 N 个已知句子（每个句子是一个"类别"）。增量训练的流程是：

1. **扩展分类头**：把输出维度从 N 扩大到 N+1，N 个旧类别的权重完整保留，新类别随机初始化
2. **重放旧数据**：从旧类别中随机抽样少量样本，和新句子数据混合训练，防止"灾难性遗忘"
3. **低学习率微调**：用比初次训练低 10–100 倍的学习率（默认 1e-4），只训练分类头，5 个 epoch 即可收敛
4. **保存部署**：覆盖 `models/sign_language/` 下的模型、LabelEncoder、LabelMap，重启后端即生效

### 前置条件

- 已至少完成一次完整训练，存在以下文件之一：
  - `model/last_checkpoint.pt`（推荐）
  - `models/sign_language/sign_language_model.pth` + `label_encoder.pkl`
- Python 环境已安装 `torch`, `torchvision`, `tqdm`, `scikit-learn`
- GPU 推荐但非必需（CPU 也能跑，只是更慢）

### 基本用法

```bash
cd D:\Aprogress\Shen\Project\Back

# 增量训练一个新句子（例如"今天天气很好"）
python scripts/incremental_train.py --sentence "今天天气很好"

# 一次训练多个句子（重复运行即可，每次新增一个）
python scripts/incremental_train.py --sentence "明天见"
python scripts/incremental_train.py --sentence "谢谢"
```

运行结束后，直接重启后端即可识别新句子：

```bash
python -m uvicorn app.main:app --host 0.0.0.0 --port 9000 --reload
```

### 参数说明

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `--sentence` | **必填** | 新增的手语句子文本 |
| `--checkpoint` | `model/last_checkpoint.pt` | 已有检查点路径 |
| `--output_dir` | `models/sign_language` | 输出（部署）目录 |
| `--replay_samples` | 500 | 旧类别重放样本数。越大越防遗忘，但训练越慢 |
| `--new_samples` | 40 | 新句子生成的哑元训练样本数 |
| `--epochs` | 5 | 增量训练轮数 |
| `--lr` | 1e-4 | 学习率（建议比初次训练低 10-100 倍） |
| `--batch_size` | 16 | 批次大小 |
| `--unfreeze_backbone` | 关闭 | 是否解冻骨干网络。开启后训练更慢但效果更好 |

### 验证结果

训练完成后，脚本会输出新增句子的标签 ID。然后你可以：

```bash
# 1. 检查模型是否已更新
python -c "
import torch
ckpt = torch.load('models/sign_language/sign_language_model.pth', map_location='cpu', weights_only=False)
print('总类别数:', ckpt['config']['num_classes'])
print('新增句子:', ckpt.get('incremental', {}).get('new_sentence', ''))
"

# 2. 调用识别 API 测试（需先启动后端）
curl -X POST "http://localhost:9000/api/v1/recognize/image" \
  -F "file=@你的手语图片.jpg"
```

### 工作原理详解

```
┌───────────────────────────────────────────────────────────┐
│  检查点 (last_checkpoint.pt)                               │
│  ├── model_state_dict: { backbone.*, classifier.* }        │
│  ├── label_encoder_classes: ["你好", "谢谢", ...]          │
│  └── config: { num_classes: N, model_name: "resnet18" }    │
└───────────────┬───────────────────────────────────────────┘
                │
    ┌───────────▼──────────┐
    │  扩展分类头           │
    │  classifier.8.weight  │  ← 从 [N, 256] 扩展到 [N+1, 256]
    │  前 N 行 = 保留       │     • 旧权重完整拷贝
    │  第 N+1 行 = Xavier   │     • 新行随机初始化
    └───────────┬──────────┘
                │
    ┌───────────▼──────────┐
    │  混合训练数据          │
    │  50% 重放旧类别        │  ← 防止灾难性遗忘
    │  50% 新句子            │
    └───────────┬──────────┘
                │
    ┌───────────▼──────────┐
    │  微调 5 epochs        │  lr=1e-4, 冻结骨干, AdamW
    │  训练集: ~540 条       │
    │  验证集: 10 条（新类别）│
    └───────────┬──────────┘
                │
    ┌───────────▼──────────┐
    │  部署文件              │
    │  sign_language_model.pth │
    │  label_encoder.pkl     │
    │  label_map.json        │
    └───────────────────────┘
```

## 方式二：RAG 视频检索索引增量更新

### 原理

RAG 检索系统基于 **句向量（Sentence Embedding）**：将所有手语句子用中文句向量模型编码为固定维度的向量，查询时用余弦相似度找到最接近的视频。

索引由两个文件组成：

| 文件 | 内容 |
|------|------|
| `embeddings_dev.npy` | N 条句子的向量矩阵 [N, D] |
| `metadata_dev.json` | N 条元数据（video_id, video_path, sentence） |

"增量更新"就是：**在原始 CSV 中添加新句子数据，然后重新构建索引**。由于编码是离线的，目前没有"追加一条到已有 numpy 矩阵"的快捷方式，推荐直接重建。

### 前置条件

- CE-CSL 数据集已下载（包含 `label/` 和 `video/` 目录）
- 安装了 `sentence-transformers`：
  ```bash
  pip install sentence-transformers
  ```
- 已有视频文件放置于正确路径下

### 基本用法

**步骤 1：准备好新句子的视频和 CSV 记录**

在 CE-CSL 数据集的 label CSV 中追加一行（以 `dev.csv` 为例）：

| Number | Translator | Chinese |
|--------|-----------|---------|
| 你的编号 | 你的子目录 | 你的新句子 |

同时将视频文件放到对应的 `video/dev/{Translator}/{Number}.mp4`。

**步骤 2：重新构建索引**

```bash
cd D:\Aprogress\Shen\Project\Back

# 重新构建 dev 索引（默认包含所有已存在 + 新增的数据）
python -m scripts.build_cecsl_index \
    --data-root "D:/Aprogress/Shen/dataset/CE-CSL/CE-CSL" \
    --split dev \
    --model BAAI/bge-small-zh-v1.5
```

这会覆盖 `data/cecsl_index/embeddings_dev.npy` 和 `metadata_dev.json`，新增的句子会被编码并加入索引。

**步骤 3：重启后端**

后端在启动时会自动加载最新的索引文件。

```bash
python -m uvicorn app.main:app --host 0.0.0.0 --port 9000 --reload
```

**步骤 4：测试检索**

```bash
curl "http://localhost:9000/api/v1/video_rag/search?query=你的新句子&top_k=5"
```

### 增量添加少量数据（进阶）

如果你只想添加很少的句子（比如 1-5 条），不想重新编码几千条已有数据，可以用这个 Python 片段手动追加：

```python
import numpy as np
import json
from sentence_transformers import SentenceTransformer

# 1. 加载已有索引
emb = np.load("data/cecsl_index/embeddings_dev.npy")
with open("data/cecsl_index/metadata_dev.json", "r", encoding="utf-8") as f:
    meta = json.load(f)

# 2. 编码新句子
model = SentenceTransformer("BAAI/bge-small-zh-v1.5")
new_sentence = "你的新句子"
new_vec = model.encode([new_sentence], normalize_embeddings=True)

# 3. 追加到矩阵
emb = np.vstack([emb, new_vec])

# 4. 追加元数据
meta["samples"].append({
    "sample_id": "custom_001",
    "split": "dev",
    "video_path": "dev/A/custom_001.mp4",
    "sentence": new_sentence,
    "gloss": "",
    "note": "手工增量添加"
})

# 5. 保存
np.save("data/cecsl_index/embeddings_dev.npy", emb)
with open("data/cecsl_index/metadata_dev.json", "w", encoding="utf-8") as f:
    json.dump(meta, f, ensure_ascii=False, indent=2)

print("增量添加完成，总样本数:", len(meta["samples"]))
```

## 对比总结

| 维度 | ResNet 增量训练 | RAG 索引增量更新 |
|------|----------------|------------------|
| 使用场景 | 让模型识别新手语句子（图像→文本） | 让系统检索新手语视频（文本→视频） |
| 所需数据 | 检查点 + 新手语句子文本 | CE-CSL 数据集 CSV + 视频文件 |
| 核心操作 | 扩展分类头 + 微调 | 重新编码句子向量 |
| 执行速度 | 5–10 分钟（GPU） | 取决于数据量，dev 约 30 秒 |
| 是否有遗忘风险 | 有（靠重放旧数据缓解） | 无（全量重建索引） |
| 脚本 | `scripts/incremental_train.py` | `scripts/build_cecsl_index.py` |
| 部署方式 | 覆盖 `models/sign_language/` 文件 | 覆盖 `data/cecsl_index/` 文件 |

## 常见问题

### Q1: 增量训练后，旧句子的识别准确率会下降吗？

会有轻微下降（通常 1-3%），但脚本内置的"重放"机制（默认 500 条旧数据混合训练）可以有效缓解。如果下降明显，可以增大 `--replay_samples`。

### Q2: 可以一次增量训练多个句子吗？

脚本当前一次只支持新增一个句子。你可以多次运行，每次传入不同的 `--sentence`。每次运行都会基于上一次的检查点继续扩展。

### Q3: 当前模型用的是哑元帧（dummy frame），增量训练有意义吗？

现阶段主要用于**验证流程正确性**。哑元帧训练的模型不具备真实识别能力。要真正实用，需要先实现视频帧提取管线（从 `.mp4` 提取关键帧 → 224×224 预处理 → 替换 `DummySignDataset`）。

### Q4: RAG 索引和 ResNet 模型有什么关系？

它们是两个独立的系统：
- **ResNet 模型**：做"图像→文本"分类，用于手语识别
- **RAG 索引**：做"文本→视频"检索，用于根据中文句子找对应手语视频

两者互不依赖，可以独立增量更新。

### Q5: 增量训练失败了怎么办？

每次运行前，脚本会在 `models/sign_language/incremental_YYYYMMDD_HHMMSS/` 下保存备份。出错时可以从备份恢复，或者用原始检查点重新开始。

### Q6: 我想用 ResNet50 而不是 ResNet18，怎么增量训练？

脚本会自动识别检查点中的 `model_name` 字段。如果你用 `train_model_final.py` 训练了 ResNet50，增量训练会自动使用 ResNet50 的 backbone。确保检查点中包含 `config.model_name: "resnet50"` 即可。
