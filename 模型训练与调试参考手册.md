## 一、训练参数速查表

增量训练（`finetune_new_word.py`）期间会打印以下指标：

| 参数 | 含义 | 好的表现 | 差的表现 |
|------|------|----------|----------|
| **Train CTC Loss** | 模型在训练集上的 CTC 损失，越低越拟合 | 持续下降，最终 < 1.0 | 震荡、不降、突然飙升 |
| **Val CTC Loss** | 模型在验证集上的 CTC 损失，衡量泛化能力 | 下降趋势，不持续上升 | 持续上升（过拟合/数据太少） |
| **Train Token Acc** | 训练集的 token 级预测准确率 | 稳步升高，> 80% | 不升或剧烈波动 |
| **Val Token Acc** | 验证集的 token 级预测准确率 | > 30% 算正常 | < 10% 且有下降趋势 |
| **Seq Acc** | 整句完全匹配的比例（更严格） | 有就行，不强制高 | — |
| **Best Val Loss** | 训练过程中出现过的最低验证损失 | 越来越低 | 卡住不动 |

### 如何判断过拟合

```
✅ 正常训练                          ❌ 过拟合

Epoch  Train   Val                  Epoch  Train   Val
  1    5.2     6.1                    1    5.0     6.2
  5    2.1     3.4                    5    1.2    10.5   ← Val 不降反升
 10    0.8     2.1                   10    0.3    16.4   ← 崩了
 20    0.3     1.5
```

> 如果 Val Loss 连续 10 个 epoch 不降反升，Early Stopping 会自动停止训练。原因通常是旧数据太少（少于全量会过拟合到训练集）。

### 关于 CTC Loss

CTC Loss 本身不是百分比，数值范围取决于序列长度和类别数，所以只看 **相对趋势**，不看绝对值。几百的 Loss 属正常，关键是 Train 和 Val 的走势。

## 二、验证模型准确度

### 2.1 命令行推理（单条）

```powershell
cd AAA

# 验证自定义词
python demo_infer.py --video "..\data\custom_videos\好_01.mp4" --checkpoint output/ctc_lstm/best_model_finetuned.pt
python demo_infer.py --video "..\data\custom_videos\好_05.mp4" --checkpoint output/ctc_lstm/best_model_finetuned.pt

# 验证 CE-CSL 旧词
python demo_infer.py --video "..\dataset\CE-CSL\video\dev\A\dev-00001.mp4" --checkpoint output/ctc_lstm/best_model_finetuned.pt
python demo_infer.py --video "..\dataset\CE-CSL\video\dev\B\dev-00010.mp4" --checkpoint output/ctc_lstm/best_model_finetuned.pt
```

### 2.2 推理输出解读

```
[Debug] T'=217, 平均blank概率=0.991, blank>0.9比例=0.991, argmax为blank比例=0.991

=== 推理结果 ===
字符序列: ['好']
预测句子: 好
```

| 字段 | 含义 |
|------|------|
| `T'` | CTC 输出的时间步数（≈ 视频帧数的 1/4） |
| `平均blank概率` | 时间步中有多少比例的模型认为"无内容"，高是正常的（大部分时间是空白帧） |
| `blank>0.9比例` | 模型十分确信为空白的时间步占比 |
| `argmax为blank比例` | 最终被判定为空白的帧占比 |
| `预测句子` | **最终识别结果**，对比 CSV 中该视频的 `Chinese Sentences` 即可判断对错 |

### 2.3 批量验证（统计准确率）

对所有自定义视频做推理并统计：

```powershell
cd AAA
python -c "
import os, sys
sys.path.insert(0, '.')
from demo_infer import load_checkpoint, build_feature_extractor, predict_from_features
from ctc_dataset import process_gloss
import pandas as pd

CKPT = 'output/ctc_lstm/best_model_finetuned.pt'
CSV = r'..\data\custom_videos\custom.csv'
FEAT_DIR = r'..\data\custom_videos\features'

model, vocab, device = load_checkpoint(CKPT)
extractor = build_feature_extractor()
df = pd.read_csv(CSV, encoding='utf-8')

correct = 0
for _, row in df.iterrows():
    number = str(row['Number'])
    translator = str(row['Translator'])
    label = str(row['Chinese Sentences'])
    fname = f'{number}_{translator}.npy'
    fpath = os.path.join(FEAT_DIR, fname)
    if not os.path.exists(fpath):
        continue
    import numpy as np
    feats = np.load(fpath)
    from demo_infer import predict_from_features
    tokens, sentence = predict_from_features(feats, model, vocab, device)
    ok = label == sentence
    if ok:
        correct += 1
    print(f'  {fname}: 预测=\"{sentence}\" 标签=\"{label}\" {\"✅\" if ok else \"❌\"}')

print(f'\n准确率: {correct}/{len(df)} ({correct*100/len(df):.1f}%)')
"
```

## 三、GPU 状态解读

### 3.1 特征提取阶段（GPU 低占正常）

```
特征提取 = CPU 解码视频 + GPU 秒算 ResNet18

CPU 重活                          GPU 轻活
┌─────────────────────┐         ┌──────────┐
│ 读视频 → 解码 → 裁剪  │  ────▶ │ ResNet18 │
│ 归一化 → 转 tensor    │  16帧  │  0.01秒   │
│ (占 85% 时间)         │        │ (占 15%)  │
└─────────────────────┘         └──────────┘
```

ResNet18 太小，GPU 多数时间在等 CPU 喂数据，**15% 占用是正常的**。

### 3.2 训练阶段（GPU 应满）

`finetune_new_word.py` 跑起来后 GPU 占用应该 > 80%。如果仍然低：
- 检查 `BATCH_SIZE` 是否太小（默认 8，可调大到 16 或 32）
- 确认 PyTorch 是 CUDA 版本：`python -c "import torch; print(torch.version.cuda)"`

## 四、环境变量管理

### 4.1 需要设的环境变量

| 变量 | 作用 | 影响范围 |
|------|------|----------|
| `CECSL_DATA_ROOT` | CE-CSL 数据集路径 | `extract_features.py` `train_lstm.py` `finetune_new_word.py` |
| `CTC_CHECKPOINT` | Web 后端使用的模型路径 | `Project/Back/app/ctc_service.py` |

```powershell
# CE-CSL 数据集路径
```

### 4.3 删除（恢复默认）

```powershell

```

## 六、调参指南

> 调参的核心原则：**根据曲线反馈，不是瞎试排列组合。**

```
观察现象 → 判断原因 → 调一个参数 → 重跑 → 再看曲线
```

**铁律：一次只改一个参数。** 一次改多个，永远不知道是谁起的作用。

---

### 6.1 `train_lstm.py` 参数全景

| 参数 | 默认值 | 作用 | 什么时候调 |
|------|--------|------|------------|
| `LR` | 1e-4 | 学习率，控制模型每次调整的步长 | 最重要，见下方 |
| `DROPOUT` | 0.2 | 随机丢弃神经元的比例，防过拟合 | Val Loss 升、Train Loss 降 |
| `WEIGHT_DECAY` | 1e-4 | 权重衰减，限制参数不要太大 | 同上 |
| `HIDDEN_SIZE` | 256 | BiLSTM 隐藏层大小，越大越能记 | Loss 都不降（欠拟合） |
| `NUM_LAYERS` | 2 | BiLSTM 层数 | 同上 |
| `BATCH_SIZE` | 16 | 每批处理的视频数 | 显存不够 ↓，显存有余 ↑ |
| `PATIENCE` | 15 | Val Loss 连续不降多少轮就停 | 停太早 ↑，停太晚 ↓ |
| `MIN_EPOCHS_NO_EARLY_STOP` | 30 | 前多少轮不检查早停 | 给模型足够的起步时间 |
| `WARMUP_EPOCHS` | 10 | 前多少轮学习率从 0 线性涨到 LR | 前期震荡时 ↑ |
| `EPOCHS` | 250 | 最大训练轮数 | 有早停兜底，一般不动 |
| `INPUT_SIZE` | 512 | ResNet 固定的特征维度 | **永远不要改** |

---

### 6.2 按现象查参数

#### 过拟合（最常见）

```
现象：Train Loss ↘  但  Val Loss ↗
意思：模型在背答案，不是在学规律
```

| 调整方向 | 参数 | 改动 |
|----------|------|------|
| 降低学习速度 | `LR` | 1e-4 → 5e-5 |
| 增强正则化 | `DROPOUT` | 0.2 → 0.4 |
| 限制参数膨胀 | `WEIGHT_DECAY` | 1e-4 → 1e-3 |
| 用更多数据 | `MAX_TRAIN_SAMPLES` | None（全量） |

#### 欠拟合

```
现象：Train Loss 很高 且  Val Loss 也很高，两者都不降
意思：模型太简单，学不会
```

| 调整方向 | 参数 | 改动 |
|----------|------|------|
| 提高学习速度 | `LR` | 1e-4 → 5e-4 |
| 增大模型容量 | `HIDDEN_SIZE` | 256 → 512 |
| 加深网络 | `NUM_LAYERS` | 2 → 3 |

#### Loss 震荡剧烈

```
现象：Loss 上下跳动，没有稳定趋势
意思：学习率太大，跳过头了
```

| 调整方向 | 参数 | 改动 |
|----------|------|------|
| 减小步长 | `LR` | 1e-4 → 5e-5 |
| 增大批次 | `BATCH_SIZE` | 16 → 32 |

#### 显存不够（CUDA out of memory）

| 调整方向 | 参数 | 改动 |
|----------|------|------|
| 减小批次 | `BATCH_SIZE` | 16 → 8 或 4 |
| 降低模型容量 | `HIDDEN_SIZE` | 256 → 128 |

---

### 6.3 调试优先级

```
第一优先：LR
  90% 的问题都是学习率不对。先调这个。

第二优先：DROPOUT / WEIGHT_DECAY
  Val Loss 上升时调。

第三优先：HIDDEN_SIZE / NUM_LAYERS
  两个 Loss 都不降时调。

最后：PATIENCE / WARMUP_EPOCHS
  控制什么时候停、起步速度。
```

## 七、常见问题

### Q: 训练完验证集 Loss 很高但推理正确？

说明模型在你的自定义词上学得很好，但 CE-CSL 旧词泛化不够（通常因为旧数据太少）。推理你自己的视频不受影响。

### Q: 怎么知道模型有没有灾难性遗忘？

对比新旧模型在 CE-CSL 视频上的推理结果：
```powershell
# 旧模型
python demo_infer.py --video "..\dataset\CE-CSL\video\dev\A\dev-00010.mp4" --checkpoint "..\Project\Back\model\last_checkpoint.pt"

# 新模型
python demo_infer.py --video "..\dataset\CE-CSL\video\dev\A\dev-00010.mp4" --checkpoint output/ctc_lstm/best_model_finetuned.pt
```

### Q: `blank 概率=0.99` 正常吗？

完全正常。CTC 模型的 blank 概率天然偏高——大多数时间步都对应"无内容"的空白帧，只有手势动作发生的几帧才会触发非 blank 输出。

### Q: 训练被 Early Stopping 中断了？

Val Loss 连续 10 轮不降就自动停。需要更多旧数据、减小学习率、或增大 `PATIENCE` 参数（在 `FinetuneConfig` 中）。
