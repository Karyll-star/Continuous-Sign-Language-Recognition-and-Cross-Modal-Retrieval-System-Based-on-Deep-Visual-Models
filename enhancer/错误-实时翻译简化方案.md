## 三、后端改动详情

### 3.1 ctc_service.py — 模型缓存 + 真实置信度

**问题：** 当前每次调用 `predict_from_image()` 都重新执行 `load_checkpoint()`，加载 ResNet18 + BiLSTM 模型（~50MB），耗时 2-5 秒。

**修复：** 添加全局缓存的模型加载器和置信度计算函数。

```python
# ========== 添加到 ctc_service.py 文件末尾 ==========

import numpy as np
import torch
from typing import Tuple

# 全局模型缓存
_cached_model = None
_cached_vocab = None
_cached_device = None
_cached_extractor = None


def get_cached_model(checkpoint_path: str = None):
    """获取缓存的模型组件（全局单例，只加载一次）"""
    global _cached_model, _cached_vocab, _cached_device, _cached_extractor

    if _cached_model is not None:
        return _cached_model, _cached_vocab, _cached_device, _cached_extractor

    if checkpoint_path is None:
        checkpoint_path = os.environ.get(
            "CTC_CHECKPOINT",
            os.path.join(
                os.path.abspath(os.path.join(os.path.dirname(__file__), "..")),
                "model", "last_checkpoint.pt"
            )
        )

    from demo_infer import load_checkpoint, build_feature_extractor

    _cached_model, _cached_vocab, _cached_device = load_checkpoint(checkpoint_path)
    _cached_extractor = build_feature_extractor()

    print(f"[ctc_service] 模型已缓存: device={_cached_device}")
    return _cached_model, _cached_vocab, _cached_device, _cached_extractor


def infer_with_confidence(
    image_path: str,
    checkpoint_path: str = None
) -> dict:
    """
    单帧推理并返回真实置信度。

    返回: { "text": str, "tokens": list, "confidence": float }
    """
    model, vocab, device, extractor = get_cached_model(checkpoint_path)

    img = cv2.imread(image_path)
    if img is None:
        raise RuntimeError(f"无法读取图片: {image_path}")

    feats = extractor.extract([img])  # (1, 512)

    # 前向传播
    feats_t = torch.from_numpy(feats).float().unsqueeze(0).to(device)
    feat_lens_t = torch.tensor([1], dtype=torch.long).to(device)
    log_probs, _ = model(feats_t, feat_lens_t)  # (1, 1, C)

    # CTC greedy 解码
    log_probs_2d = log_probs.squeeze(1)  # (1, C)
    best_idx = log_probs_2d[0].argmax().item()

    if best_idx == 0:  # blank
        return {"text": "", "tokens": [], "confidence": 0.0}

    token = vocab.id2token.get(best_idx, "<unk>")

    # 置信度 = softmax 概率 × 100
    probs = torch.softmax(log_probs_2d[0], dim=-1)
    confidence = round(float(probs[best_idx].item()) * 100, 1)

    return {
        "text": token,
        "tokens": [token],
        "confidence": confidence,
    }
```

**要点：**
- `get_cached_model()` — 首次调用时加载模型，后续直接返回缓存引用
- `infer_with_confidence()` — 替代旧的 `recognize_image_file()`，单次前向传播同时完成解码和置信度计算
- 置信度 = 预测 token 在 softmax 中的概率值 × 100（0-100）

```python
# 构建查表字典（DICTIONARY_DATA 后添加）
_TEXT_LOOKUP = {}
for _item in DICTIONARY_DATA:
    _key = _item.get("chinese", "").strip()
    if _key:
        _TEXT_LOOKUP[_key] = {
            "pinyin": _item.get("pinyin", ""),
            "meaning": _item.get("meaning", ""),
        }

def lookup_text_info(text: str) -> dict:
    text = text.strip()
    if text in _TEXT_LOOKUP:
        return _TEXT_LOOKUP[text]
    for kw, info in _TEXT_LOOKUP.items():
        if kw in text:
            return info
    return {"pinyin": "", "meaning": ""}
```

## 五、完整数据流

```
摄像头 (30fps)
  │  每 1000ms 采样一帧
  ▼
captureFrame() → JPEG base64 (quality=0.8)
  │
  ▼  HTTP POST
POST /recognize/upload
Content-Type: multipart/form-data
{ file: <frame.jpg> }
  │
  ▼  main.py
1. 写入临时文件
2. 调用 ctc_service.infer_with_confidence(tmp_path)
   ├── get_cached_model() → 缓存命中（首次则加载）
   ├── extractor.extract([img]) → (1, 512)
   ├── model.forward() → log_probs (1, 1, C)
   ├── greedy argmax → token
   └── softmax[token] → confidence (0-100)
3. lookup_text_info(token) → { pinyin, meaning }
4. 返回 JSON
  │
  ▼  HTTP 响应
{
  "code": 200,
  "data": {
    "results": [{
      "text": "你好",
      "pinyin": "nǐ hǎo",       ← 不再是空字符串
      "meaning": "表示问候或打招呼", ← 不再是空字符串
      "confidence": 87.3        ← 真实 softmax 概率
    }]
  }
}
  │
  ▼  前端 recognize.vue
recognitionStore.setResult({ text, pinyin, meaning, confidence })
→ 结果面板实时展示
```

## 七、可选升级：HTTP 帧缓冲

如果单帧识别准确率不满足需求，可以在简化方案基础上做**小幅升级**：

### 思路

后端加一个 `/recognize/realtime` 端点，服务端维护帧缓冲区：

```
POST /recognize/realtime
Body: { "session_id": "abc123", "image": "<base64>" }

服务端:
  buffer[session_id].append(frame)
  if len(buffer[session_id]) >= 8:
      feats = extractor.extract(buffer[session_id])
      result = model.forward(feats)  → 多帧 CTC 推理
      buffer[session_id] = buffer[session_id][-8:]  # 滑动窗口
      return result
  else:
      return { "status": "buffering", "count": len }
```

```
POST /recognize/realtime/reset
Body: { "session_id": "abc123" }

服务端: del buffer[session_id]
```

### 改动量

| 文件 | 改动 |
|------|------|
| `ctc_service.py` | 新增 `SessionBuffer` 类（~40 行） |
| `main.py` | 新增 2 个 HTTP 端点（~60 行） |
| `recognize.vue` | 改 URL、传 session_id（~10 行） |

总增量约 **110 行**，仍然远少于 WebSocket 方案的 200 行。

### 何时升级

- 单帧识别频繁返回空或错误结果时
- 用户反馈"识别不准"时
- 需要识别连续手语动作（多词序列）时

---

> **建议路径：** 先实施简化方案（本文档），快速上线验证效果。如果单帧准确率不达标，再按第七章升级到 HTTP 帧缓冲。
