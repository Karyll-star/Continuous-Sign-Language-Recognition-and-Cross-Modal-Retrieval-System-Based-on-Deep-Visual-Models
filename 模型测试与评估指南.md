## 二、单条推理测试

### 命令

```powershell
cd AAA

# 模型 A：新模型（增量训练产物）
python demo_infer.py --video "..\data\custom_videos\好_01.mp4" --checkpoint output/ctc_lstm/best_model_finetuned.pt

# 模型 B：旧模型（项目默认）
python demo_infer.py --video "..\data\custom_videos\好_01.mp4" --checkpoint "..\Project\Back\model\last_checkpoint.pt"

# CE-CSL 任意视频
python demo_infer.py --video "..\dataset\CE-CSL\video\dev\A\dev-00001.mp4" --checkpoint output/ctc_lstm/best_model_finetuned.pt
```

### 输出解读

```
使用视频推理: ..\data\custom_videos\好_01.mp4
[Debug] T'=217, 平均blank概率=0.991, blank>0.9比例=0.991, argmax为blank比例=0.991

=== 推理结果 ===
字符序列: ['好']
预测句子: 好
```

| 字段 | 含义 | 正常范围 |
|------|------|----------|
| `T'` | CTC 输出的时间步数 | 视频帧数的约 1/4 |
| `平均blank概率` | 多少帧被判定为"无内容" | 0.9+ 正常（大部分是空白帧） |
| `blank>0.9比例` | 模型十分确信为空白的帧占比 | 接近 blank 概率 |
| `argmax为blank比例` | 最终判定为 blank 的帧占比 | 接近 blank 概率 |
| `预测句子` | **核心结果**，对比视频对应的 CSV 标注 | 匹配即正确 |

### 怎么判断对错

1. 找到视频的 CSV 标注行，看 `Chinese Sentences` 列
2. 对比 `预测句子` 是否一致
3. 不一致不代表模型毁了——可能是这个特定视频本来就难

> **重点**：单条结果有偶然性，批量评估才反映真实水平。

### 3.3 指标含义

| 指标 | 含义 | 合理范围 |
|------|------|----------|
| **样本数** | 验证集视频总数 | 515（全量） |
| **CTC Loss** | 验证损失，越低越好 | 1-5 正常 |
| **Token Acc** | 逐词预测正确率 | 30-60% |
| **Seq Acc** | 整句完全匹配率 | 5-20%（手语翻译任务本身就低） |
| **Loss 变化** | 新旧模型 Loss 差值 | < 0.5 = 优秀，0.5-1 = 正常，> 1 = 退化 |

### 3.4 判断结论

```
Loss 变化 < 0     → ✅ 新模型更好（微调提升了泛化）
Loss 变化 0~0.5   → ✅ 增量训练未伤害旧词
Loss 变化 0.5~1.0 → ⚠️ 轻微退化，可接受
Loss 变化 > 1.0   → ❌ 灾难性遗忘，需跑全量特征重做增量训练
```

## 四、自定义词准确率统计

批量统计你所有自定义视频的识别率：

```powershell
cd AAA
python -c "
import os, numpy as np, pandas as pd
from demo_infer import load_checkpoint, predict_from_features

CKPT = 'output/ctc_lstm/best_model_finetuned.pt'
CSV = r'..\data\custom_videos\custom.csv'
FEAT_DIR = r'..\data\custom_videos\features'

model, vocab, device = load_checkpoint(CKPT)
df = pd.read_csv(CSV, encoding='utf-8')
correct = 0
total = 0

for _, row in df.iterrows():
    fname = f'{row.Number}_{row.Translator}.npy'
    fpath = os.path.join(FEAT_DIR, fname)
    if not os.path.exists(fpath):
        continue
    total += 1
    feats = np.load(fpath)
    _, sentence = predict_from_features(feats, model, vocab, device)
    label = str(row['Chinese Sentences'])
    ok = label == sentence
    if ok:
        correct += 1
    print(f'  {row.Number}: 预测=\"{sentence}\" 标签=\"{label}\" {\"✅\" if ok else \"❌\"}')

print(f'\n自定义词准确率: {correct}/{total} ({correct*100/total:.1f}%)')
"
```

### 预期结果

| 准确率 | 含义 |
|--------|------|
| > 90% | ✅ 非常好，模型充分掌握了你的手势 |
| 70-90% | ⚠️ 可用，建议增加视频数量或多样性 |
| < 70% | ❌ 需要改进——检查视频质量、增加样本量 |

## 七、常见测试问题

### Q: 单条推理错误，是不是模型毁了？

不一定。拿同一条视频去跑旧模型也错的话，说明这条视频本身就难——可能是光照差、手势不清晰、或者是低频词。批量评估才说明问题。

### Q: Token Acc 只有 30%，是不是很差？

手语翻译任务中 30-50% Token Acc 属正常范围。Seq Acc（整句全对）通常只有 5-20%。关键是新旧模型差距不要太大。

### Q: 新模型 Loss 比旧模型高怎么办？

- 差距 < 1.0 → 可接受，自定义词识别不受影响
- 差距 > 1.0 → 跑全量特征 `--max-samples 0` 重做增量训练

### Q: 怎么只评估新模型（不对比旧模型）？

`eval_models.py` 会自动检测文件是否存在的，删除或重命名旧模型文件它就只评估新模型。

```powershell
python eval_models.py   # 自动跳过不存在的模型
```

### Q: 想限制评估样本数？

修改 `eval_models.py` 中的 `MAX_SAMPLES = None` 为具体数字（如 `MAX_SAMPLES = 100`）。

### Q: 测试视频需要提前提取特征吗？

是的。`eval_models.py` 和 `demo_infer.py` 都依赖预提取的特征（`.npy` 文件）。如果没有：
```powershell
python extract_features.py --splits dev --max-samples 0
```
