### 第 1 步：准备数据（已完成）

**CE-CSL 数据集** 放在 `dataset/CE-CSL/`，包含 `label/` 和 `video/` 两个文件夹。

**自定义数据** 放在 `data/custom_videos/`：
- 视频：`*.mp4`（每个词至少 10 段）
- 标注：`custom.csv`
- 特征：`features/`（已通过 `extract_custom_features.py` 生成）

> `custom.csv` 格式示例：
> ```csv
> Number,Translator,Chinese Sentences,Gloss,video_file,Note
> 00001,Z,好,好,好_01.mp4,
> ```


### 第 2 步：提取 CE-CSL 全量特征

```powershell
cd AAA

# 1. 设环境变量
$env:CECSL_DATA_ROOT = (Resolve-Path "..\dataset\CE-CSL").Path

# 2. 提取全量特征（4973 条 train + 515 条 dev，约 1-2 小时 GPU）
python extract_features.py --splits train,dev --max-samples 0
```

> **快速验证**：先把 `--max-samples 0` 改成 `--max-samples 500` 跑小批量验证流程，确认无误后再跑全量。
> 
> **注意**：特征提取阶段 GPU 占用 15% 左右是正常的——ResNet18 太小，大部分时间花在 CPU 解码视频上。训练阶段 GPU 会拉满。

特征保存到：
- `dataset/CE-CSL/train_features/`
- `dataset/CE-CSL/val_features/`


### 第 3 步：增量训练

```powershell
cd AAA
python finetune_new_word.py
```

脚本依次执行：

1. 从 `custom.csv` 提取新词
2. 加载预训练模型 `Project/Back/model/last_checkpoint.pt`（71MB）
3. 扩展词表（若新词已在词表中则跳过）
4. 扩展 fc 层（旧权重保留，新维度随机初始化）
5. 构建混合数据集：CE-CSL 训练集 2000 条 + 自定义数据
6. 加权采样微调：新样本权重 ×5，学习率 1e-4，最多 30 轮

输出模型：`output/ctc_lstm/best_model_finetuned.pt`。

#### 训练参数解读

| 参数 | 含义 | 好的表现 | 差的表现 |
|------|------|----------|----------|
| **Train CTC Loss** | 训练集损失 | 持续下降 → < 1.0 | 震荡不降 |
| **Val CTC Loss** | 验证集损失 | 下降趋势，不持续上升 | 持续上升（过拟合） |
| **Train Token Acc** | 训练集准确率 | > 80% | 不升 |
| **Val Token Acc** | 验证集准确率 | > 30% | < 10% 且下降 |

#### 判断过拟合

```
✅ 正常                           ❌ 过拟合

Epoch  Train   Val               Epoch  Train   Val
  1    5.2     6.1                 1    5.0     6.2
  5    2.1     3.4                 5    1.2    10.5   ← Val 反升
 10    0.8     2.1                10    0.3    16.4   ← 崩了
```

> Val Loss 连续 10 轮不降 → Early Stopping 自动停。原因：旧数据太少。跑全量特征可解决。

#### GPU 状态

训练阶段 GPU 占用应 > 80%。特征提取阶段 15% 正常（CPU 解码视频是瓶颈）。


### 第 4 步：测试验证

#### 单条推理

```powershell
# 自定义词
python demo_infer.py --video "..\data\custom_videos\好_01.mp4" --checkpoint output/ctc_lstm/best_model_finetuned.pt

# CE-CSL 旧词
python demo_infer.py --video "..\dataset\CE-CSL\video\dev\A\dev-00001.mp4" --checkpoint output/ctc_lstm/best_model_finetuned.pt
```

#### 输出解读

```
[Debug] T'=217, 平均blank概率=0.991, blank>0.9比例=0.991, argmax为blank比例=0.991
=== 推理结果 ===
字符序列: ['好']
预测句子: 好
```

| 字段 | 含义 |
|------|------|
| `预测句子` | 识别结果，对比 CSV 中的 `Chinese Sentences` |
| `平均blank概率` | CTC 认为"无内容"的帧占比，0.9+ 完全正常 |

#### 批量统计准确率

```powershell
cd AAA
python -c "
import os, numpy as np, pandas as pd
from demo_infer import load_checkpoint, predict_from_features

CKPT = 'output/ctc_lstm/best_model_finetuned.pt'
CSV = r'..\data\custom_videos\custom.csv'
FEAT_DIR = r'..\data\custom_videos\features'

model, vocab, device = load_checkpoint(CKPT)
df = pd.read_csv(CSV, encoding='utf-8')
correct = 0
for _, row in df.iterrows():
    fpath = os.path.join(FEAT_DIR, f'{row.Number}_{row.Translator}.npy')
    if not os.path.exists(fpath): continue
    _, sentence = predict_from_features(np.load(fpath), model, vocab, device)
    ok = str(row['Chinese Sentences']) == sentence
    if ok: correct += 1
    print(f'  {row.Number}: 预测=\"{sentence}\" 标签=\"{row[\"Chinese Sentences\"]}\" {\"✅\" if ok else \"❌\"}')
print(f'\n准确率: {correct}/{len(df)} ({correct*100/len(df):.1f}%)')
"
```

## 模型切换（Web 后端）

增量训练不影响前端默认模型。要用新模型识别，通过环境变量切换：

```powershell
cd Project/Back

# 切换到增量训练模型
$env:CTC_CHECKPOINT = "..\..\AAA\output\ctc_lstm\best_model_finetuned.pt"
python -m uvicorn app.main:app --host 0.0.0.0 --port 9000 --reload

# 切回默认模型
Remove-Item Env:\CTC_CHECKPOINT
# 重启服务即可
```

> 原理：`ctc_service.py` 优先读 `CTC_CHECKPOINT` 环境变量，不存在时回退到 `Project/Back/model/last_checkpoint.pt`。

## 常见问题

### Q1: 运行脚本时路径错误，提示 `D:\Aprogress\Shen\...` 不存在？

环境变量没设。先执行 `$env:CECSL_DATA_ROOT = (Resolve-Path "..\dataset\CE-CSL").Path`。

### Q2: 特征提取时 GPU 占用只有 15%？

正常。ResNet18 太小，瓶颈在 CPU 解码视频。训练阶段 GPU 会拉满。

### Q3: 训练 Val Loss 很高但自定义词推理正确？

自定义词识别不受影响。Val Loss 高是因为验证集是 CE-CSL 旧词，旧数据覆盖不全导致泛化不够。你的视频推理正常即可。

### Q4: 预训练模型报错 "invalid header or archive is corrupted"？

`Project/Back/model/last_checkpoint.pt` 是有效的 71MB 文件。如果损坏，`git checkout` 恢复即可。`AAA/output/ctc_lstm/best_model.pt` 也是损坏的，不要用它。

### Q5: `[跳过] 词汇 'X' 已存在词表中`？

说明这个词已经在 CE-CSL 预训练模型的 3853 个词里了。增量训练仍然会跑，效果是微调模型适配你的手势风格。

### Q6: 全量特征提取太慢？

正常。4973 条视频逐帧提取特征约 1-2 小时。可以后台跑，或者先用 `--max-samples 500` 快速验证。

### Q7: 以后加新词怎么做？

1. 录制新视频放入 `data/custom_videos/`
2. 更新 `custom.csv` 加入新词
3. `python extract_custom_features.py`
4. `python finetune_new_word.py`

CE-CSL 特征无需重新提取。

### Q8: 怎么对比新旧模型的差异？

```powershell
# 旧模型
python demo_infer.py --video "..\dataset\CE-CSL\video\dev\A\dev-00010.mp4" --checkpoint "..\Project\Back\model\last_checkpoint.pt"

# 新模型
python demo_infer.py --video "..\dataset\CE-CSL\video\dev\A\dev-00010.mp4" --checkpoint output/ctc_lstm/best_model_finetuned.pt
```
