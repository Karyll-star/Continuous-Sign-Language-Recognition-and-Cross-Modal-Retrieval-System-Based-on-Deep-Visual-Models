# 增量训练指南 —— 让模型识别自定义词汇

> 在已训练好的手语识别模型基础上，添加对新词（如"拿铁"）的识别能力，而不会影响原有词汇。

---

## 概述

本项目的 CTC 手语识别模型（`AAA/` 目录下的 ResNet18 + 1D-CNN + BiLSTM + CTC 架构）默认只能识别训练数据集中出现过的词汇。当你需要模型识别一个数据集中不存在的新词时，需要通过 **增量训练（Finetune）** 来扩展模型的词汇表和识别能力。

### 适用场景

- 需要识别 CE-CSL 数据集中不存在的专有名词（如"拿铁"、"咖啡"、"WiFi"等）
- 需要识别自定义的手语手势
- 需要为特定业务场景定制识别词汇

### 不适用场景

- 仅想在词典中查询某个词的手语视频 → 直接在 `Project/Back/app/main.py` 的 `DICTIONARY_DATA` 中添加即可，无需训练

### 第 1 步：录制视频并创建标注文件

#### 1.1 创建目录结构

在项目根目录下创建 `data/custom_videos/`，将视频和标注文件都放在这里：

```
项目根目录/
└── data/
    └── custom_videos/           ← 你的自定义数据
        ├── 拿铁_01.mp4          ← 手语视频（gitignore 已忽略）
        ├── 拿铁_02.mp4
        ├── 拿铁_03.mp4
        ├── ...（10段以上）
        ├── custom.csv           ← 标注文件
        └── features/            ← 自动生成（第2步），gitignore 已忽略
            ├── 00001_Z.npy
            └── ...
```

> `.gitignore` 已配置忽略 `*.mp4` 和 `*.npy`，放心放在这里，不会误提交。

#### 1.2 录制视频的要求

| 项目 | 建议 |
|------|------|
| 时长 | 2-5 秒/段 |
| 数量 | **至少 10 段**（越多越好，推荐 20-50 段） |
| 分辨率 | 不做强制要求，ResNet 会统一缩放到 224×224 |
| 格式 | `.mp4`（H.264 编码） |
| 背景 | 尽量简洁，避免复杂背景干扰特征提取 |
| 角度 | 不同角度和速度各录几段，提高鲁棒性 |
| 手势 | 建议参考《国家通用手语词典》的标准手势 |

> **数据量与准确率的关系**：
> - 1-5 段：❌ 很差，模型几乎学不到有效特征
> - 10-20 段：⚠️ 基本可用，开始学到手势的共性
> - 20-50 段：✅ 推荐范围，泛化能力较好
> - 50+ 段：✅ 边际递减，继续增加仍有益但提升幅度变小
>
> **多样性比纯粹的数量更关键**：2 个不同演示者 > 1 个演示者录 50 段。

**【推荐】扁平模式**——包含 `video_file` 列直接指定视频文件名：

```csv

00002,Z,拿铁,拿铁,拿铁_02.mp4,
00003,Z,拿铁,拿铁,拿铁_03.mp4,
00004,Z,拿铁,拿铁,拿铁_04.mp4,
00005,Z,拿铁,拿铁,拿铁_05.mp4,
00006,Z,拿铁,拿铁,拿铁_06.mp4,
00007,Z,拿铁,拿铁,拿铁_07.mp4,
00008,Z,拿铁,拿铁,拿铁_08.mp4,
00009,Z,拿铁,拿铁,拿铁_09.mp4,
00010,Z,拿铁,拿铁,拿铁_10.mp4,
```

字段说明：

| 字段 | 说明 | 示例 |
|------|------|------|
| `Number` | 唯一标识（建议用序号） | `00001` |
| `Translator` | 演示者标识（可自定义） | `Z` |
| `Chinese Sentences` | 对应的中文句子/词语 | `拿铁` |
| `Gloss` | 手语词序列（用 `/` 分隔多个词） | `拿铁` |
| `video_file` | **视频文件名**（相对于 data/custom_videos/） | `拿铁_01.mp4` |
| `Note` | 备注（可选） | 自定义 |

> **同时添加多个新词**：直接在同一 CSV 中添加多行，不同 `Gloss` 值对应不同新词即可。脚本会自动发现所有新词并一起扩展。
>
> **兼容层级模式**：如果你更喜欢 CE-CSL 的层级目录结构（`data/custom_videos/Z/00001.mp4`），也可以省略 `video_file` 列，脚本会自动按层级查找。

## 进阶配置

### 修改超参数

所有可调参数都在 `finetune_new_word.py` 的 `FinetuneConfig` 类中：

```python
@dataclass
class FinetuneConfig:
    # 新数据太少时，增大此值让新词被更频繁采样
    NEW_SAMPLE_WEIGHT: float = 5.0   # 默认 5.0，可调到 10.0

    # 新词收敛不充分时，增大这些值
    EPOCHS: int = 30                  # 默认 30，可调到 50
    LR: float = 1e-4                  # 默认 1e-4

    # 担心遗忘旧词时，增大抽样量
    MAX_OLD_TRAIN_SAMPLES: int = 2000 # 默认 2000，可调到 5000
```

## 技术原理

### 为什么不能直接加词？

CTC（Connectionist Temporal Classification）模型的输出层是一个全连接层：

```
fc: Linear(hidden_size * 2, num_classes)
```

`num_classes` 在模型初始化时就固定了，等于词表大小。添加新词意味着 `num_classes` 需要增加，而新的权重维度必须通过训练获得。

### 为什么不能只训练新词？

如果只用新词数据微调，模型会迅速 **忘记旧词**（灾难性遗忘），因为：

- 新词数据只有 10 条 vs 旧词数据 2000+ 条
- 梯度更新会覆盖对所有旧词有用的参数

所以我们必须 **混合旧数据 + 新数据**，让模型同时看到两者。

### fc 层扩展策略

```
旧 fc.weight: (2000, 512)          新 fc.weight: (2001, 512)
┌─────────────────────┐            ┌─────────────────────────┐
│ 词0 (blank)  ████████│            │ 词0 (blank)  ████████    │ ← 保留
│ 词1 (unk)    ████████│            │ 词1 (unk)    ████████    │ ← 保留
│ ...          ████████│   ──────▶  │ ...          ████████    │ ← 保留
│ 词1999       ████████│            │ 词1999       ████████    │ ← 保留
└─────────────────────┘            │ 词2000(拿铁)  ░░░░░░░░    │ ← 随机初始化
                                    └─────────────────────────┘
```

新增维度的初始化为旧权重的均值 + 小噪声，这样初始输出不会偏向任何类别。

## 常见问题

### Q1: 视频播放正常但 `extract_custom_features.py` 报"找不到视频"

确认 `video_file` 列的值与 `data/custom_videos/` 下的文件名完全一致。如果你用的是层级模式（无 `video_file` 列），确保目录结构为 `data/custom_videos/Z/00001.mp4`。

### Q2: `finetune_new_word.py` 报"预训练模型不存在"

说明基础模型尚未训练。先执行：
```bash
python extract_features.py --splits train,dev,test --max-samples 0
python train_lstm.py
```

### Q3: 微调后旧词识别变差了

增大 `MAX_OLD_TRAIN_SAMPLES`、降低 `NEW_SAMPLE_WEIGHT`，或降低 `LR` 重新微调。

### Q4: 新词识别准确率仍然很低

- 增加录制视频的数量和多样性（不同角度、速度、光照）
- 确保手势清晰、符合标准手语规范
- 增大 `EPOCHS` 和 `NEW_SAMPLE_WEIGHT`

### Q5: 想连词成句（如"我要一杯拿铁"）

CTC 模型本身支持连续识别——当视频中包含多个手势时，它会按时间顺序输出多个 Gloss 词。你只需要在 `Gloss` 字段中标注序列即可：

```csv
00011,Z,我要一杯拿铁,我/要/一/杯/拿铁,拿铁_sentence.mp4,
```

但需要注意，组成句子的**每个 Gloss 词都必须在词表中**。如果某些基础词（如"我"、"要"）已包含在 CE-CSL 数据集中，它们会自动存在于词表中。

### Q6: 我有 GPU，但脚本用了 CPU

检查 PyTorch 是否检测到 CUDA：
```bash
python -c "import torch; print(torch.cuda.is_available())"
```
如果输出 `False`，请安装 CUDA 版本的 PyTorch。

### Q7: pip install 缺少依赖

两个脚本依赖：`torch`, `torchvision`, `opencv-python`, `pandas`, `numpy`, `tqdm`。如果提示缺少某个包，手动安装即可：
```bash
pip install torch torchvision opencv-python pandas numpy tqdm
```
