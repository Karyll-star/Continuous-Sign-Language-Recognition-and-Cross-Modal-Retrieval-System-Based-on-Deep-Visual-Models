## 二、详细工作

### 2.1 CE-CSL 数据集配置与验证

原开源项目需要 CE-CSL（Chinese Sign Language）数据集才能完成完整训练，但数据路径硬编码且不易获取。

**完成工作**：
- 成功网上检索并下载完整的 CE-CSL 数据集（含训练集/验证集/测试集的视频与标注）
- 逐一验证数据集与原项目期望格式的兼容性（CSV 列名、演示者目录结构、数据字段）
- 通过 `CECSL_DATA_ROOT` 环境变量解耦硬编码路径，提升脚本可移植性
- 确认数据集规模：训练集约 4973 条 + 验证集约 515 条

### 2.2 "无声咖啡馆"场景模型增量训练

为适配"无声咖啡馆"的具体需求，在预训练模型基础上新增咖啡相关术语的识别能力。

**完成工作**：
- 设计并跑通**增量训练（Finetune）**完整流程：
  1. 采集自定义手语视频（每个词 ≥ 10 段，多角度多速度）
  2. 使用冻结的 ResNet18 提取 512 维视觉特征
  3. 基于 CTC Loss 扩展词表并混合训练（自定义词 ×5 加权采样）
  4. 批量测试验证新旧词识别准确率
- 新增咖啡术语词汇：**拿铁、美式** 等
- 通过混合抽样策略有效防止"灾难性遗忘"（旧词准确率不退化）

**技术方案**：提供了两条路径：
- **方案 A（独立训练）**：无需 CE-CSL 数据集，仅针对自定义词快速训练
- **方案 B（增量训练）**：基于 CE-CSL 全量 + 自定义数据，同时保留旧词能力

### 2.3 实时翻译能力修复与优化

原开源项目存在严重的实时翻译缺陷：WebSocket 未完整实现、实际为逐张拍照翻译（缺失动作时序信息）、置信度展示仅起装饰作用。

**完成工作**：
- 修复 WebSocket 后端服务，实现真正的 **连续视频帧级实时识别**
- 打通"摄像头采集 → WS 帧传输 → 后端 CTC 推理 → 前端实时展示"全链路
- 完善 WebSocket 消息协议：`auth` / `frame` / `start` / `stop` / `ping` / `result` / `error` / `pong`
- 优化前端识别结果的置信度展示逻辑，使其反映真实模型输出

### 2.4 LLM 后处理优化（简化版）

针对 CTC 模型输出碎片化、存在识别错误的问题，设计并实现了基于云端大语言模型的后处理增强方案。**方案文档位于 `enhancer/` 目录，实际功能代码位于另一 Git 分支中**。

**已实现方案（简化版 v2.0）**：

- **双通道架构**：快通道保持现有词级实时推送不变，慢通道对累积的词语序列周期性调用 LLM 进行纠错与成句
- **TokenBuffer 上下文累积**：滑动窗口维护最近 N 个 CTC token，支持可替换的触发策略（当前为每 3 个 token 触发一次）
- **LLM Gateway 抽象层**：统一接口支持阿里百炼 Qwen-Plus API（当前实现）和本地 Ollama Qwen2.5（预留接口），可热切换与自动降级
- **Prompt 工程**：设计专门的手语后处理 Prompt——纠正常见识别错误（如"尺饭"→"吃饭"）、去除重复词、将碎片组装为流畅中文句子
- **最小化侵入**：所有新代码集中在 `enhancer/` 目录（约 155 行），现有 `main.py` 仅增加 1 个 import + 1 行调用

**enhancer 目录技术文档**：

| `enhancer/LLM后处理优化技术架构方案.md` | 完整三阶段架构设计（词语纠错 + 句子组装 + 拼音补全），含状态机和网关抽象 |
| `enhancer/LLM后处理简化方案.md` | v2.0 落地实施方案，含完整代码（TokenBuffer、Prompt、Gateway） |
| `enhancer/错误-实时翻译简化方案.md` | v2.1 最小改动方案（模型缓存 + 真实置信度 + 词典查表） |

### 2.5 前端展示页面精简

本项目实际定位为手语翻译智能眼镜的**云端计算程序**——眼镜端提供视频流，云端程序返回识别文本与语音结果。因此前端仅需保留核心展示功能，对原有 Web 页面进行了大幅精简。

**完成工作**：
- **删除冗余页面与功能**：移除登录/注册、翻译历史、个人中心、管理后台、词典搜索等面向 C 端用户的页面和相关路由
- **仅保留核心展示页**：实时翻译页面 `recognize.vue` 作为唯一对外界面，直接进入视频实时翻译流程
- **配色体系重构**：从原有深色/紫色风格，统一为**蓝绿色光谱渐变**（teal + sky + lime），营造"科技温度"的品牌调性
- **代码精简**：移除认证 Store、收藏 Store、路由鉴权中间件等不再需要的模块，减少维护负担

## 三、技术要点总结

### 3.1 模型架构

```
输入视频 (T 帧)
    │
    ▼
┌─────────────────┐
│  ResNet18        │  空间特征提取（ImageNet 预训练，冻结）
│  输出: (B,T,512) │
└─────────────────┘
    │
    ▼
┌─────────────────┐
│  1D-CNN          │  局部时序建模
│  Conv1d+MaxPool  │  时间维下采样
└─────────────────┘
    │
    ▼
┌─────────────────┐
│  BiLSTM (2层)    │  全局时序建模
│  Hidden: 256     │  输出: (B, T', 512)
└─────────────────┘
    │
    ▼
┌─────────────────┐
│  CTC Classifier  │  序列对齐与解码
│  (Greedy Decode) │  输出: 中文文本
└─────────────────┘
```

### 3.2 增量训练策略

- 词表扩展 + FC 层扩展（旧权重保留，新维度随机初始化）
- 混合数据集：CE-CSL 训练集 2000 条 + 自定义数据
- 加权采样：新样本 ×5 权重，防止被大量旧数据淹没
- 低学习率微调（1e-4），最多 30 轮，Early Stopping 自动终止

### 3.3 技术栈

| 层级 | 技术 |
|------|------|
| 模型框架 | PyTorch 2.1.0 |
| 视觉骨干 | ResNet18（ImageNet 预训练） |
| 时序建模 | 1D-CNN + 双向 LSTM |
| 对齐损失 | CTC Loss（无需帧级标注） |
| 后端服务 | FastAPI + WebSocket |
| 前端框架 | Nuxt 3 + Vue 3 + TypeScript |
| 样式系统 | Tailwind CSS |
| 文本向量化 | BGE-small-zh-v1.5（384 维） |

## 四、原项目问题修复清单

在集成过程中发现并修复了原开源项目的若干问题：

| # | 问题描述 | 修复方案 |
|---|----------|----------|
| 1 | 后端文件索引使用硬编码绝对路径 | 引入 `CECSL_DATA_ROOT` / `CTC_CHECKPOINT` 环境变量 |
| 2 | 前端环境配置缺失 | 补充 `.env` 配置及启动文档 |
| 3 | 实时翻译实为逐张拍照，WebSocket 未完工 | 重写 WebSocket 服务端，实现连续帧级识别 |
| 4 | 置信度展示为固定值，不反映实际模型输出 | 接入真实 CTC 输出概率 |
| 5 | 摄像头调用失败 | 修复 `useCamera` composable 的浏览器兼容性 |
| 6 | "开始识别"按钮被视频区域遮挡 | 调整前端 z-index 与布局 |
| 7 | 注册后退出再登录提示"未注册" | 修复 `auth.ts` 中密码字段在登录时被覆盖的 bug |
| 8 | 历史记录缩略图使用 `blob:` URL，刷新后失效 | 改用后端稳定 URL 或图标占位降级 |

## 五、后续优化方向

1. **LLM 后处理完整版上线**：在已实现的简化版基础上，将 `enhancer/` 分支中的完整三阶段 LLM 管道（词语纠错 + 句子组装 + 拼音/释义自动补全）合并到主分支，支持静默超时/置信度骤降等高级句子边界检测策略，以及本地 Ollama + 云端 API 混合部署模式
2. **新词扩展**：继续为"无声咖啡馆"场景录制更多手语视频（如"卡布奇诺""摩卡""牛奶""糖"等），扩充模型词表
3. **语音输出**：集成 TTS 语音合成，使听障人士的翻译结果可被语音播放，实现双向无障碍沟通
4. **模型压缩**：通过量化/剪枝等手段减小模型体积，降低推理延迟
